# Noey & Top Private Space

แอปเว็บเล็กๆ สำหรับเก็บความทรงจำของเรา สร้างด้วย React + Vite + Tailwind CSS + DaisyUI เก็บข้อมูลฝั่งเบราว์เซอร์ (localStorage) ใช้งานง่าย และรองรับมือถือ

## ฟีเจอร์หลัก

- Diary: เขียนบันทึกพร้อมหัวข้อ อารมณ์ และเนื้อหา แสดงเวลาที่บันทึก และลบรายการได้
- Star Jar: จดข้อความสั้นๆ ที่อยากขอบคุณ/ประทับใจ จัดเรียงรายการล่าสุดก่อน และลบได้
- Countdowns: เพิ่มเหตุการณ์พร้อมวันเวลา แสดงเวลานับถอยหลังแบบเรียลไทม์ (วัน ชั่วโมง นาที วินาที) และขึ้นป้ายว่าเกิดขึ้นแล้วเมื่อถึงเวลา ลบได้
- Photo Board: ติดบอร์ดรูปภาพด้วยการวาง URL พร้อมคำบรรยาย แสดงเป็นกริด และลบรูปได้
- Missions: สร้างภารกิจเล็กๆ ทำเครื่องหมายสำเร็จ/ไม่สำเร็จได้ รายการที่สำเร็จจะแสดงเส้นขีดทับ และลบได้
- นำทางสะดวก: แถบนำทางด้านบน (Navbar) พร้อมเมนูแบบ dropdown สำหรับจอเล็ก
- รองรับมือถือ: แบบฟอร์มบนจอเล็กจะเรียงแนวตั้งและปุ่มเต็มความกว้าง กริดรายการปรับจำนวนคอลัมน์ตามขนาดหน้าจอ
- ข้อมูลอยู่กับเครื่อง: ทุกอย่างบันทึกใน localStorage ไม่ต้องมีเซิร์ฟเวอร์

## เริ่มต้นใช้งาน

ต้องมี Node.js เวอร์ชัน 18 ขึ้นไป

- `npm install` — ติดตั้งแพ็กเกจ
- `npm run dev` — เปิดเซิร์ฟเวอร์สำหรับพัฒนา
- `npm run build` — สร้างไฟล์สำหรับขึ้นโปรดักชัน
- `npm run preview` — พรีวิวไฟล์ที่ build แล้ว

เปิดเบราว์เซอร์ไปที่ที่อยู่ที่ dev server แสดง (เช่น `http://localhost:5173`)

## เทคโนโลยีที่ใช้

- React + Vite
- Tailwind CSS + DaisyUI (ธีมกำหนดเองชื่อ `noeyTop` ดูใน `tailwind.config.cjs`)
- React Router v6 (กำหนดเส้นทางหน้าเพจ)
- localStorage สำหรับเก็บข้อมูล (`src/lib/storage.ts`)

## โครงสร้างโปรเจกต์

- `src/components/` — `Navbar`, `DiaryCard`, `StarCard`, `PhotoItem`
- `src/pages/` — `Home`, `Diary`, `StarJar`, `Countdown`, `PhotoBoard`, `Missions`
- `src/styles/index.css` — ตั้งค่า Tailwind และสไตล์พื้นหลังนุ่มๆ

## ธีมและสไตล์

กำหนด `data-theme="noeyTop"` ไว้บนแท็ก `html` เพื่อใช้ธีมโทนพาสเทล สามารถปรับสีได้ใน `tailwind.config.cjs`

## การเก็บข้อมูลและความเป็นส่วนตัว

- ข้อมูลทั้งหมดถูกเก็บไว้ในเบราว์เซอร์ของผู้ใช้ผ่าน localStorage เท่านั้น ไม่ถูกส่งไปยังเซิร์ฟเวอร์
- การลบแคช/ข้อมูลเว็บไซต์หรือเปลี่ยนเบราว์เซอร์/เครื่อง อาจทำให้ข้อมูลหาย

## ข้อจำกัดปัจจุบัน

- Photo Board รองรับการเพิ่มรูปผ่าน URL เท่านั้น (ยังไม่รองรับอัปโหลดไฟล์โดยตรง)
- ไม่มีการซิงก์ข้อมูลข้ามอุปกรณ์/เบราว์เซอร์
- เมื่อกดลบข้อมูลแล้วจะไม่สามารถกู้คืนได้

## เคล็ดลับการใช้งาน

- หากต้องการสำรองข้อมูลแบบง่าย สามารถคัดลอกค่าจาก `localStorage` ผ่าน DevTools เพื่อนำไปเก็บไว้ แล้ววางคืนภายหลังได้
